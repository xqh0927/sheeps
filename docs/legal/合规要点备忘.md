# 秘境消消乐 · 法律合规要点备忘

> 本文档随两份协议（隐私政策.md、用户协议.md）一并生成，记录起草依据、关键义务与**需要运营方补充/确认的事项**。
> 生成日期：2026-07-07 ｜ 适用口径：中国大陆（PIPL 为主线）

## 一、起草依据（适用法规）

| 法规 | 对应义务 |
| --- | --- |
| 《个人信息保护法》(PIPL) | 告知-同意、最小必要、第17条隐私政策必备条款、第44-50条用户权利、出境合规 |
| 《网络安全法》 | 实名要求、日志留存 ≥6 个月、安全保护义务 |
| 《数据安全法》 | 数据分类分级、重要数据保护 |
| 《未成年人保护法》+ 防沉迷规定 | 实名注册、时长/时段限制、未成年人消费限制 |
| 《网络出版服务管理规定》/ 游戏版号 | 商用网络游戏须取得版号与出版许可（如面向公众运营） |

## 二、本项目已核实的事实（决定协议内容）

- **无**第三方广告/统计/推送/支付 SDK（libs.versions.toml 仅含基础库）。
- 采集字段：手机号（只读）、密码（PBKDF2 哈希，非明文）、昵称、头像（Base64≤512KB）、游戏进度/积分/任务、设备型号/系统/IP/日志。
- 通信全量 **AES-256-GCM** 加密；服务端 **Cloudflare Workers + D1**。
- **无内购**，积分为纯虚拟权益，不可变现。
- 多语言资源（en/ja/ko/zh-rTW）存在 → 可能服务海外用户，但默认按中国法起草。

## 三、必须补充的占位信息（协议中以【】标注）

1. **运营主体全称、注册地址**（两份协议均需）。
2. **个人信息保护负责人邮箱、客服邮箱/渠道**（隐私政策第五、八节；用户协议第十四节）。
3. **短信服务商名称**（隐私政策附录 B；建议补全具体厂商如阿里云/腾讯云短信）。
4. **账号注销后数据留存期**（建议 6–12 个月，需与备份策略核对）。
5. **争议管辖法院**（用户协议 12.2，填运营主体住所地）。

## 四、上线前必须满足的合规动作（风险清单）

| 优先级 | 事项 | 依据 |
| --- | --- | --- |
| 高 | 在 App 首次启动时以**弹窗**形式取得隐私政策同意（单独同意，不能默认勾选） | PIPL 第14、17条 |
| 高 | 提供**便捷的账号注销**入口（设置-账号与安全） | PIPL 第47条 |
| 高 | 落实**个人信息权利响应通道**（查阅/更正/删除/撤回/转移） | PIPL 第44-50条 |
| 高 | 若作为商用网络游戏：**取得版号 + 接入国家实名认证 + 防沉迷系统** | 未成年人保护法、版号规定 |
| 中 | 若数据出境（Cloudflare 全球节点）：履行**出境合规**（标准合同备案/安全评估） | 个人信息出境标准合同办法 |
| 中 | 安全事件 **72 小时内** 报告主管部门并通知用户 | 网络安全法、PIPL |
| 中 | 日志留存 ≥6 个月、建立访问审计 | 网络安全法 |
| 低 | 如未来新增广告/统计/支付 SDK：更新协议并重新取得同意 | PIPL 第23、25条 |

## 五、建议（非必须但推荐）

- 将两份协议渲染为 **HTML** 放入 App WebView，便于展示与版本管理；当前为 Markdown 源。
- 建立协议**版本号 + 更新日志**，便于合规追溯。
- 若计划出海（en/ja/ko 等），需分别做 GDPR/日本 APPI/韩国 PIPA 等本地化合规评估，不应仅用中文版翻译。
