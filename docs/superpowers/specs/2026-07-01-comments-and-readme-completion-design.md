# 代码注释汉化与 README 补全设计方案

本项目是一个端云协同的中式民俗传说主题三消手游（类似于“羊了个羊”）。本方案旨在对整个项目的 116 个源文件（包含 Android 客户端和 Cloudflare Workers 后端）进行地毯式中文注释补全与翻译，同时更新和完善项目根目录及 Android 客户端的 `README.md`。

## 用户审查与确认

本项目的所有英文注释均需修改为对应的中文注释，并补全缺失的方法、类和复杂步骤注释。由于改动范围巨大，开发工作将按照模块依赖顺序（方案 1）分阶段迭代修改并进行编译验证。

## 设计规范

### 1. 代码注释规范
* **翻译一致性**：将全部英文注释汉化，保留英文技术专有名词（如 `WebSocket`、`Room`、`Retrofit`、`LCG`、`D1`、`Canvas`）。
* **注释补全深度**：
  * **文件与类头**：补全清晰的类或对象职责描述、使用生命周期、主要依赖等。
  * **方法与函数**：补全方法主要流程、参数（`@param`）及返回值（`@return`）的说明。
  * **行内核心步骤**：为复杂的数学公式（10%面积重叠公式、对决连击伤害）及复杂的控制流程补全行内说明。
* **排版细节**：在中文字符和代码变量/英文单词之间增加半角空格，句末统一使用中文标点符号。

### 2. 两个 README.md 补全规划

#### 客户端 `app/README.md` 补全
* **Mermaid 拓扑图**：绘制清晰的模块间依赖关系图（`:app`、`:core`、各个 `:feature` 模块）。
* **MVI 实战指南**：以游戏界面为例展示 `State` ⇄ `Intent` ⇄ `Effect` 的数据流转模式。
* **路由索引表**：详细列出 `TheRouter` 的所有注册路由 URI，对应的页面与参数。
* **Room 本地同步**：详细阐述 `SyncRepository` 在 Room 本地离线脏数据与云端同步的机制。

#### 根目录 `README.md` 补全
* **系统架构图**：绘制整体端云协同架构。
* **算法核心公式**：
  * 10% 重叠遮挡的几何公式及对齐算法。
  * 连击对决伤害缓动公式 $Attack = Base \times (1 + \ln(Combo))$。
  * LCG 种子全局类型（40%封印、20%盲盒、40%正常）及端云一致性保障。
* **一键发布流水线**：详细梳理 `release.js` 的工作机制，CI 触发过滤条件，以及 APK 的 HEAD 探针延迟发布缓存检测流程。

## 任务执行拆分（方案 1）

* **阶段一**：后端 `server/src/` 源码文件（共 14 个）的中文注释补全与翻译。
* **阶段二**：客户端 `:core` 基础公共模块（共 41 个）的 Kotlin 代码中文化。
* **阶段三**：客户端 `:feature_game` 游戏主模块（共 27 个）的 Kotlin 代码中文化。
* **阶段四**：客户端 `:feature_menu` 及其它业务 UI 模块（共 34 个）的代码中文化。
* **阶段五**：更新和补全项目根目录与 Android 客户端的 `README.md`。

## 验证计划

### 自动化构建验证
在完成每一阶段的修改后，运行编译命令以确保代码无任何编译报错：
* 后端校验：`cd server; npm run build`
* 前端校验：`./gradlew.bat compileDebugKotlin`
