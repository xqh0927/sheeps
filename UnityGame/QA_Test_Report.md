4. **⚠️ 槽位插入顺序问题** ❌

   - **影响**: 槽位中卡牌的排列顺序与Android版本不一致，可能影响用户体验

## Bug列表

### P0（严重）- 0个

无

### P1（重要）- 1个

#### Bug #1: 槽位插入顺序不一致

- **描述**: Unity版本直接将卡牌添加到槽位末尾，Android版本将卡牌插入到相同类型的最后一张后面
- **位置**: `GameManager.ClickTile()` 方法
- **影响**: 槽位中卡牌的排列顺序与Android版本不一致，可能影响用户体验
- **修复建议**:
```csharp
// 在 GameManager.ClickTile() 中，替换：
slotData.AddTile(tile);

// 改为：
InsertIntoSlot(tile);

// 新增方法：
private void InsertIntoSlot(Tile newTile)
{
    int lastIndex = slotData.tiles.FindLastIndex(t => t.type == newTile.type);
    if (lastIndex == -1)
    {
        slotData.tiles.Add(newTile);
    }
    else
    {
        slotData.tiles.Insert(lastIndex + 1, newTile);
    }
}
```

- **描述**: 网络失败时，分数应该本地保存，下次联网时同步
- **位置**: `ApiManager.SubmitScore()` 方法
- **影响**: 网络不稳定时，用户分数可能丢失
- **修复建议**: 在 `SubmitScore()` 中实现本地缓存逻辑，在网络恢复时同步

#### Bug #3: 卡牌消除动画未实现

- **描述**: 卡牌消除时，没有缩放/淡出动画
- **位置**: `SlotUI.RemoveTileFromSlot()` 方法
- **影响**: 用户体验不佳，缺少视觉反馈
- **修复建议**: 在 `RemoveTileFromSlot()` 中实现缩放/淡出动画
